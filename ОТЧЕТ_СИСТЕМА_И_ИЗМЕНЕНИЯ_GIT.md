## 1. Резюме для заказчика

**calc.figurebase.ru** — это веб-система учёта и аналитики результатов соревнований по фигурному катанию на основе расчётных XML (ISU CalcFS и совместимые выгрузки): публичный просмотр, закрытие данных для гостей при включённом пароле, администрирование импорта, богатый блок **аналитики** (бесплатное участие, сегменты школ МАФКК/ЦСКА, новички, PDF), **JSON API** для интеграций, выгрузки в **Google Sheets**, набор **офлайн-скриптов** для чистки справочников.

В **данном** Git-репозитории история начинается с **24.01.2026** (initial уже «полная версия»); по состоянию на дату отчёта зафиксировано **156 коммитов** эволюции. Ориентир замены стоимости разработки текущего функционала — **порядка 2,5–6,5 млн ₽** и **5–9 человеко-месяцев** (см. разделы 3–4 и 21).

## 3. Методика оценки стоимости эволюции по коммитам

Оценка **не является коммерческим предложением**. Для каждого месяца: трудозатраты в часах ≈ `N_коммитов × 1,2` (нижняя граница) и `× 3,5` (верхняя), стоимость — часы × **2000 ₽/ч** и × **3500 ₽/ч**. Так отражается смесь мелких правок и крупных фич без разбора каждого тикета. Для месяца с **initial «полная версия»** (январь) эта формула **занижает** фактическую трудоёмкость первой поставки — см. раздел 21.

## 5. Назначение, архитектура и стек

- **Назначение:** единая точка для хранения турниров, спортсменов, клубов, тренеров, выступлений, судейских оценок и производных отчётов (в т.ч. политика бесплатного участия, сегментация школ, новички).  
- **Паттерн:** монолитное Flask-приложение через **application factory** (`app_factory.py`), blueprints `public`, `admin`, `analytics`, `api`, обработчики ошибок `routes/errors.py`.  
- **ORM:** Flask-SQLAlchemy + **Flask-Migrate** (Alembic).  
- **Шаблоны:** Jinja2 + статика `static/css/style.css`.  
- **Безопасность:** Flask-WTF **CSRF** (API blueprint exempt с отдельной аутентификацией), **Flask-Limiter** (при `REDIS_URL` — хранение в Redis, иначе in-memory), **ProxyFix** за nginx, `validate_security_at_startup`.  
- **Интеграции:** Google Sheets (`google_sheets_sync.py`), внешние вызовы из скриптов.

## 6. Конфигурация и эксплуатация

Ключевые параметры из `config.py` и окружения (неполный перечень): `SECRET_KEY`, `SQLALCHEMY_DATABASE_URI` / `DATABASE_URL`, `UPLOAD_FOLDER`, `MAX_CONTENT_LENGTH`, `ADMIN_USERNAME`, `ADMIN_PASSWORD` / `ADMIN_PASSWORD_HASH`, `SITE_READ_PASSWORD`, **`API_KEYS`**, `LOG_LEVEL`, `LOG_FILE`, `RATE_LIMIT_PER_MINUTE`, `SESSION_COOKIE_SECURE`, `CORS_ORIGINS` (в `extensions.init_cors` — только для `/api/*`). Рекомендуется прод: отдельный пользователь ОС, ротация логов, бэкапы SQLite/Postgres по регламенту, переменные секретов только из окружения.

## 7. Роли, доступ и безопасность

### 7.1. Гость

При заданном `SITE_READ_PASSWORD` и включённом HTML-gate: страницы blueprints **public** и **analytics** с персональными/турнирными данными требуют сессии читателя или админа; редирект на `/site-access?next=…`, заголовок `Cache-Control: private, no-store`. Главная `/` без редиректа.

### 7.2. Читатель («доступ судьи»)

Пароль на `/site-access`; опционально журнал `SiteReaderLoginLog` (дата, IP, User-Agent) и админ-шаблон просмотра.

### 7.3. Администратор

`/admin/login`, полный доступ к импорту, нормализации, отчётам, журналам, Google Sheets.

### 7.4. API

Сессия админа/читателя или Bearer/`X-API-Key` из `API_KEYS`, либо `DISABLE_PUBLIC_API_AUTH` только для dev.

## 13. Импорт XML и связанные сервисы

Цепочка: загрузка файла → подготовка/анализ (`services/xml_import_prepare.py`, `import_service.py`) → нормализация категорий в UI → при необходимости проверка конфликтов дат рождения (`import_birth_conflict.py`, POST `/check-import-birth-conflicts`) → загрузка в БД → **архив XML** (`xml_archive.py`). Парсер: `parsers/isu_calcfs_parser.py`.
