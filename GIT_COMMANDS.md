# Команды для работы с Git

## 🔄 Полная синхронизация (отправка + обновление)

### Вариант 1: Автоматический скрипт (рекомендуется)
```bash
python sync_with_git.py
```

### Вариант 2: Ручное выполнение команд

#### ШАГ 1: Отправка изменений с сервера на Git

```bash
# 1. Проверить статус
git status

# 2. Добавить все изменения
git add .

# 3. Создать коммит
git commit -m "Обновление от $(date)"

# 4. Отправить на git
git push
```

#### ШАГ 2: Обновление локального проекта с Git

```bash
# 1. Получить информацию об обновлениях
git fetch

# 2. Обновить локальную ветку
git pull
```

## 🔍 Полезные команды

### Проверить статус
```bash
git status
```

### Посмотреть историю коммитов
```bash
git log --oneline -10
```

### Проверить настройки remote репозитория
```bash
git remote -v
```

### Если нужно добавить/изменить remote
```bash
# Посмотреть текущие remote
git remote -v

# Добавить remote (если его нет)
git remote add origin <URL_ВАШЕГО_РЕПОЗИТОРИЯ>

# Изменить URL remote
git remote set-url origin <НОВЫЙ_URL>
```

### Если есть конфликты при pull
```bash
# Посмотреть конфликты
git status

# После разрешения конфликтов вручную:
git add .
git commit -m "Разрешение конфликтов"
git push
```

---

## ⚠️ Важные замечания

1. **Перед push всегда проверяйте статус**: `git status`
2. **Не коммитьте чувствительные данные**: пароли, ключи API (они в .gitignore)
3. **Если push не работает**, проверьте:
   - Настроен ли remote: `git remote -v`
   - Есть ли права доступа к репозиторию
   - Правильно ли указан URL репозитория
